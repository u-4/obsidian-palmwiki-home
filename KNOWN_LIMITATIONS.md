# 既知の制限

- 0.5.0はObsidian Desktop 1.12.7以上を正式な対象とし、macOS版1.12.7で確認しています。Windows / Linuxは未確認です。
- 0.5.0のレスポンシブUIは、端末名ではなく各Home／Card領域の実幅でヘッダー折り返しとSquare列数を決め、タッチ向けの操作領域も備えています。iPhone / iPad実機で表示と基本操作を試験していますが、iOS版Obsidianでの起動復元、回転、Split View、バックグラウンド復帰、ソフトウェアキーボード、長時間利用時のメモリは確認中です。公開版は引き続きデスクトップ対応です。
- Squareの2列境界はカード領域のCSS pxであり、端末の物理解像度ではありません。既定480はiPhone縦画面を2列に保つ推奨値です。大きく設定しすぎるとiPadやデスクトップでも2列が続き、小さくしすぎると狭い画面のカードがCard size別配置へ移るため、通常は既定値を使用してください。
- `Square`カードは限られた高さで本文と画像を優先するため、パス、PageRank / In / Out、タグ、フォルダー、更新日を意図的に表示しません。これらが必要な場合は`Portrait`またはTableを使用してください。
- iOS / Androidは正式対応外です。iOSでは一部の実機確認を開始していますが、上記のライフサイクル確認が完了するまで対応環境とは表示しません。Androidは未確認です。
- 本文検索はローカルの文字列一致とVault内リンクを対象とし、OCR、添付ファイル本文、複数Vault、AIによる意味検索、同義語の自動展開は行いません。空白区切り語はAND条件で、1回の検索は最大256文字・除外語を含む合計8語です。超過時は検索を実行せず説明を表示します。
- 検索順位は本文・ページ名・別名・タグ・パスの一致、直接リンク、最大2-hopのリンク、既存のPageRank風スコアを組み合わせた説明可能な近似値です。Cosenseの非公開アルゴリズムと同一ではなく、リンクが少ないVaultや巨大hubでは期待と異なる場合があります。
- 画面停止を防ぐため、関連度計算は検索語1語ごとに直接リンク20,000辺、2-hop 50,000辺までです。非常に密なリンク構造では、パス順に決定的に打ち切るため一部の関連候補を省略する場合があります。
- 2-hopだけでつながり、検索語の本文一致・ページ名一致・直接リンクのいずれもないページは、ノイズ抑制のため結果へ追加しません。
- 入力中の候補はページ名、ファイル名、別名だけを曖昧検索します。本文検索は候補を選ばずEnterで確定した後に実行します。
- 検索結果の順位は正規化済み検索本文から計算します。画面内に入った結果のスニペットと、結果を開く時の一致位置は現在の原文を必要なページだけ再読込して求めるため、ノートを編集した直後は表示が短時間で更新される場合があります。
- Live Preview / Source modeでは一致箇所を選択できます。Reading viewには任意の描画済み文字列を選択する安定した公開APIがないため、一致行への移動だけを可能な範囲で試み、強調表示は保証しません。
- 検索状態はObsidianの戻る・進む用のworkspace状態に保存される場合があります。左上HomeボタンでHomeへ戻る場合は検索状態を復元しません。
- PageRankはGoogle PageRankそのものではなく、Vault内のリンクと更新日時を利用した静的な重要度指標です。
- PageRankは解決済みのMarkdown間リンクだけを使用します。タグ、未解決リンク、Markdown以外の埋め込み、OCR本文は対象外です。
- PageRankの除外パターンが正規表現として不正な場合は部分一致として扱われ、設定画面にはエラーを表示しません。
- 除外設定はPageRankへの影響だけを止めます。表示上の被リンク数とリンク数は変わりません。
- フォルダー設定はテキスト入力で、フォルダーピッカーはありません。
- タグ絞り込みは完全一致です。
- テーブル行は仮想化の安定性を優先して固定高のため、長い説明や多数のタグは省略表示されます。
- 保存済み索引を先に表示するため、`Waiting`または`Indexing`の間は変更前の情報が一時的に見える場合があります。`Complete`で最新状態への検証が完了します。
- 初回利用時、キャッシュ破損時、対象設定の変更時には索引を作り直すため、Vaultの規模に応じた待ち時間が発生します。
- 表示用索引と本文メタデータに加え、正規化済みMarkdown全文を含む`search-cache.json`をVault内のプラグイン専用フォルダーへ保存します。外部送信はありませんが、`.obsidian`を同期・バックアップする仕組みはこのファイルも扱う場合があります。メモリ保護のため、対象は1ファイル8 MiB、原文合計64 MiB、推定RAM 128 MiBまでです。正規化で文字数が増える破損キャッシュも保持前に上限判定し、超過時は部分索引を公開せずinclude / excludeを狭める案内を表示します。64 MiBを超える全文検索キャッシュは永続保存しません。
- 約7,000ページのコピーVaultで表示速度と基本操作を確認していますが、検索の体感速度と順位品質は本文総量とVaultのリンク構造に左右されます。検索結果は最初の100件を表示し、最大500件まで100件ずつ追加できます。それ以上は検索語やフィルターで絞り込みます。
- 本文検索キャッシュの差分再利用はVault相対パス、更新時刻、ファイルサイズがすべて同じことを確認します。外部ツールが更新時刻とサイズを意図的に同一のまま本文だけを書き換えた場合は検知できないため、`Refresh`で全再構築してください。
- 左上Homeボタン、Vault相対パス、通常Markdownの中央検索欄、中央タイトルと重複するパンくずの非表示にはObsidianの公開された専用ヘッダーAPIがないため、`.view-header-left`、`.view-header-title-container`、`.view-header-title`、`.view-header-title-parent`へ限定的に依存します。設置に失敗した場合は標準タイトルを残しますが、Obsidian側のヘッダー構造が将来変わった場合は互換更新が必要です。
- 狭い分割ペインではVault相対パスを省略表示します。完全なパスはカーソルを置いた時のツールチップとアクセシビリティ名に残ります。
- Obsidianコマンドの一覧取得と実行は、公開された`App`型に含まれないcommand managerを実行時に安全確認して利用します。非対応のObsidian版、無効化されたコマンド、現在のタブで実行できないコマンドでは、画面を変えずNoticeを表示します。
- Obsidian起動直後でまだ読み込まれていない非アクティブな復元タブでは、手入力した相対ページ・ローカル見出し・Markdown文脈依存コマンドが、そのタブの読み込み完了まで利用できない場合があります。ページ選択画面で保存した正確なVault相対パスには影響しません。
- Card previewはObsidian標準の「ページプレビュー」コアプラグインを利用します。コアプラグインが無効な場合は表示されません。また、Obsidian側でPalmWiki Home cardsの修飾キー設定を変更した場合は、その設定が最終的な表示条件になります。
- Card preview元の解除は公開型に含まれないworkspace互換処理を実行時確認して使用します。非対応版でもカードクリックには影響せず、プレビュー通知を停止して安全に終了します。
- PalmWiki Homeは既存MarkdownをObsidian公開APIの`WorkspaceLeaf.openFile()`で開き、workspaceイベントを手動で再発火しません。新しく作られるMarkdown画面へDOMを注入する他プラグインは、自身の注入先DOMが準備できるまで独自に待機する必要があります。
