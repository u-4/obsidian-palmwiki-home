# Security Policy

## Supported versions

セキュリティ修正の対象は、原則として最新のGitHub Releaseです。古い版を使用している場合は、まず最新版で再現するか確認してください。

## Reporting a vulnerability

脆弱性は公開Issueではなく、[GitHubの非公開脆弱性報告](https://github.com/u-4/obsidian-palmwiki-home/security/advisories/new)から報告してください。非公開報告が利用できない場合も、再現手順や秘密情報を公開Issueへ書かず、リポジトリ管理者へ連絡してください。

報告には、可能な範囲で次を含めてください。

- PalmWiki Home、Obsidian、OSのバージョン
- 影響する操作と再現手順
- 期待される動作と実際の動作
- 影響の範囲と、回避策の有無

Vault本文、個人情報、アクセストークン、秘密鍵、非公開ファイルは送らないでください。ログやスクリーンショットは必ず匿名化してください。

報告を受けた後は、影響の確認、修正版の準備、必要に応じたGitHub Security Advisoryでの公開を調整します。対応時間を保証するものではありませんが、受領後に確認状況を返します。
